Projects

Every repo with its servers, databases and containers. Start, stop or restart single items or whole projects; hide idle ones — they reappear the moment an agent starts them.

Access

Google accounts with access to this Console or a particular assigned domain. Configured owners always retain full access.

Incoming invites

Verified Google accounts requesting this Console or one exact private domain. Approval grants only the requested destination.

Telegram bots

Your bots deliver coordinator events for exact assigned projects. A Telegram user must send /start and be approved before receiving anything.

Servers

Dev servers managed by the coordinator and web servers running in Docker containers, grouped by repo. Each one's public subdomain is managed right here; click a row for logs and details.

Routes

Every public URL on this box. Server-backed routes are easiest to manage as the server's subdomain on the Servers page; fixed-port routes are created here.

Docker

Containers on this machine. Click a row for logs.

Port leases

Ports reserved through the coordinator, so tools and agents never collide on a port.

Pinned ports

Each dev server permanently owns its port — pins survive stops and restarts, and no other project can take a pinned port until it is unassigned.

Performance

This machine's overall health first — CPU, memory, storage, load and uptime — then live CPU and memory per running server and container. Sampled continuously by the console; history resets when the console restarts.

Usage by project

CPU and memory per project (servers + containers).

Add Google account

Invite a Gmail or Google Workspace account, then choose exactly which private domains it may open.

Allowed destinations

No selection is allowed; the account remains invited but cannot open a destination until you grant one.

Register Telegram bot

Paste a token created with BotFather. It is validated by Telegram, stored only in private server state, and never shown again.
