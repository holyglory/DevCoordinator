# DC-2026-07-16-ROUTES-01 — Docker routes select the current live replacement

## Supporting record

Production `https://skydive.vr.ae` returned the Console's styled 502 even
though the current SkydiveLive application container and local health endpoint
were healthy. Normalized coordinator inventory retained an older stopped
container before the running same-name replacement, while the Console used
`Array.find()` and therefore selected the stopped record.

The route schema intentionally uses Docker name plus container-side port so an
ordinary container replacement can retain its public hostname. Binding the
route to the immutable container ID would defeat that behavior. Removing
historical coordinator rows would also contradict the retained-state model.
The resolver therefore ranks all same-name records by lifecycle: running,
paused, then stopped or unobserved. `sampled_at` breaks ties within a lifecycle
class. Port validation still applies to the selected live identity, so a stale
stopped mapping cannot mask a missing publish on the current container.

Regression coverage first reproduced the failure with a stopped record before
a running replacement and proved the pre-fix resolver returned
`container is not running`. The adjacent control keeps a stale stopped port
from becoming a fallback when the live replacement does not publish the
requested port. The focused test and the complete DevOps Console suite passed
after the fix: 137 tests, 0 failures.

Deployment used the production `devops-console.service` restart path after the
ordinary coordinator client correctly refused mutation across the Console's
capability-asymmetric procfs boundary. The unit's layout preflight and
registration `ExecStartPost` gate passed for the new MainPID, coordinator
inventory showed the existing server and lease linked to that PID, and public
checks returned HTTP 200 for both `https://console.vr.ae/healthz` and
`https://skydive.vr.ae/health`. The unauthenticated admin route reached
SkydiveLive and returned its expected application-level HTTP 401 rather than
the Console 502.
